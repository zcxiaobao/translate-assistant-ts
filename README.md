# 职能沟通翻译助手

一个基于 AI 的翻译助手，帮助产品经理和开发工程师更好地理解彼此，弥合沟通鸿沟。

## 功能说明

### 核心功能

1. **产品 → 开发翻译**
   - 将产品经理的需求描述翻译成开发工程师可理解的技术语言
   - 自动补充技术实现方案、数据来源、性能要求和工作量评估

2. **开发 → 产品翻译**
   - 将开发工程师的技术方案翻译成产品经理可理解的业务语言
   - 自动补充用户体验影响、业务增长空间和商业价值

3. **流式输出**
   - 实时展示 AI 生成的翻译结果
   - 无需等待全部内容生成，提升用户体验

4. **智能补充**
   - 当输入内容较为模糊时，AI 会主动补充缺失的关注点和问题
   - 帮助双方更全面地理解对方的需求

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 配置 API Key

复制 `.env.example` 为 `.env` 并配置你的 OpenAI API Key：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-3.5-turbo
PORT=3000
MAX_CONTENT_LENGTH=2000
```

### 3. 运行项目

**开发模式（支持热重载）**：

```bash
npm run dev
```

**生产模式**：

```bash
npm run build
npm start
```

### 4. 访问应用

打开浏览器访问：`http://localhost:3000`

## 测试用例

### 示例 1: 产品 → 开发翻译

**输入（产品视角）**：
```
我们需要一个智能推荐功能，提升用户停留时长
```

**翻译结果（开发视角）**：
```
## 技术实现方案
建议采用协同过滤算法（Collaborative Filtering），可以选择基于用户的
协同过滤或基于物品的协同过滤。考虑到实时性要求，推荐使用 Redis 缓存
热门推荐结果，降低响应延迟。

## 数据来源和处理方式
- 数据来源：用户行为日志（浏览、点击、收藏等）
- 离线处理：每天凌晨通过 Spark 批处理计算相似度矩阵
- 实时处理：用户请求时从 Redis 读取预计算结果，结合实时行为微调

## 性能和实时性要求
- 响应时间：< 100ms
- 并发支持：QPS 1000+
- 准确率目标：推荐点击率提升 20%

## 预估开发工作量
- 算法实现和调试：3-5 天
- 数据管道搭建：2-3 天
- 接口开发和测试：2 天
- 总计：约 7-10 天

## 需要明确的技术问题
1. 用户行为数据的历史积累量有多少？
2. 是否需要考虑冷启动问题（新用户/新内容）？
3. 推荐结果的更新频率要求是实时还是准实时（如每小时更新）？
```

## 提示词设计说明

### 设计理念

本项目的核心是**提示词工程**，通过精心设计的提示词让 AI 理解产品经理和开发工程师的思维差异。

### 关键设计点

#### 1. 系统角色定义

- **产品 → 开发翻译**：AI 扮演"资深技术专家"角色
  - 关注点：技术实现、数据来源、性能要求、工作量评估
  - 语言风格：技术导向，包含专业术语

- **开发 → 产品翻译**：AI 扮演"产品顾问"角色
  - 关注点：用户体验、业务增长、商业价值、成本收益
  - 语言风格：业务导向，避免技术术语

#### 2. 结构化输出

提示词明确要求 AI 按照固定的结构输出翻译结果：
- 产品 → 开发：技术方案 → 数据处理 → 性能要求 → 工作量 → 待明确问题
- 开发 → 产品：用户体验 → 业务增长 → 商业价值 → 待关注问题

#### 3. 主动补充机制

提示词指导 AI 在输入内容不够明确时，主动补充对方角色可能关心的问题：
- 产品需求模糊时 → 补充技术关注点
- 技术方案缺少业务背景时 → 补充业务价值分析

#### 4. Few-shot Learning

每个翻译方向都提供了完整的示例，帮助 AI 理解期望的输出风格和内容深度。

#### 5. 长度控制

提示词要求输出长度在 200-500 字，确保信息量充足但不冗长。

### 提示词迭代建议

如需优化提示词效果，可以：
1. 调整系统角色的描述，使其更贴近实际工作场景
2. 增加或修改 Few-shot 示例，覆盖更多场景
3. 细化结构化输出格式，使其更符合团队需求
4. 根据实际使用反馈，调整关注点清单

## 技术架构

### 后端

- **语言**：TypeScript
- **框架**：Express
- **AI SDK**：OpenAI
- **流式输出**：Server-Sent Events (SSE)

### 前端

- **技术栈**：原生 HTML/CSS/JavaScript
- **通信**：Fetch API + ReadableStream
- **样式**：响应式设计，支持移动端

### 项目结构

```
src/
  ├── services/
  │   ├── ai-provider.ts      # AI 提供商接口
  │   ├── openai-provider.ts  # OpenAI 实现
  │   └── translator.ts       # 翻译服务
  ├── prompts/
  │   ├── pm-to-dev.ts        # 产品→开发提示词
  │   └── dev-to-pm.ts        # 开发→产品提示词
  ├── routes/
  │   └── translate.ts        # 翻译路由
  ├── app.ts                  # Express 应用
  └── server.ts               # 服务器入口
public/
  └── index.html              # Web UI
```
