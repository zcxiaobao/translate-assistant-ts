## MODIFIED Requirements

### Requirement: 提供翻译方向选择
用户界面必须提供翻译方向选择功能，允许用户从 6 个翻译方向中选择（产品↔开发、产品↔运营、开发↔运营）。

#### Scenario: 用户选择产品到开发翻译
- **WHEN** 用户在下拉菜单中选择"产品→开发"选项
- **THEN** 系统标记翻译方向为"产品→开发"，并准备接收产品需求描述

#### Scenario: 用户选择开发到产品翻译
- **WHEN** 用户在下拉菜单中选择"开发→产品"选项
- **THEN** 系统标记翻译方向为"开发→产品"，并准备接收技术方案描述

#### Scenario: 用户选择产品到运营翻译
- **WHEN** 用户在下拉菜单中选择"产品→运营"选项
- **THEN** 系统标记翻译方向为"产品→运营"，并准备接收产品需求描述

#### Scenario: 用户选择开发到运营翻译
- **WHEN** 用户在下拉菜单中选择"开发→运营"选项
- **THEN** 系统标记翻译方向为"开发→运营"，并准备接收技术方案描述

#### Scenario: 用户选择运营到产品翻译
- **WHEN** 用户在下拉菜单中选择"运营→产品"选项
- **THEN** 系统标记翻译方向为"运营→产品"，并准备接收运营需求描述

#### Scenario: 用户选择运营到开发翻译
- **WHEN** 用户在下拉菜单中选择"运营→开发"选项
- **THEN** 系统标记翻译方向为"运营→开发"，并准备接收运营需求描述

#### Scenario: 下拉菜单显示描述文案
- **WHEN** 用户打开翻译方向下拉菜单
- **THEN** 系统为每个选项显示简短的描述文案，帮助用户理解各翻译方向的用途

#### Scenario: 默认选中产品到开发方向
- **WHEN** 用户首次访问界面
- **THEN** 系统默认选中"产品→开发"翻译方向，保持与原有行为一致
