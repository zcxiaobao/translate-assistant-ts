## Why

为了提高翻译助手的灵活性和成本效益，需要支持多种大语言模型提供商。当前系统仅支持单一模型，无法根据不同场景（如成本、性能、可用性）灵活切换。通过添加模型切换功能，并默认使用 DeepSeek AI（高性价比的国内大模型），可以降低运营成本并提升系统的可维护性。

## What Changes

- 新增大语言模型提供商管理功能，支持多个 LLM 服务商
- 新增模型配置管理，包括 API Key、Base URL、模型名称等
- 新增模型切换接口，允许在不同 LLM 提供商之间切换
- 将 DeepSeek AI 设置为默认模型提供商
- 配置默认的 DeepSeek API Key
- 支持运行时动态切换模型，无需重启服务

## Capabilities

### New Capabilities

- `llm-provider-management`: 大语言模型提供商管理能力，包括提供商注册、配置、切换和调用的统一接口

### Modified Capabilities

- `translation-engine`: 翻译引擎需要支持从 LLM 提供商管理模块获取模型实例，而不是直接硬编码单一模型

## Impact

**代码影响**:
- 需要创建新的 LLM 提供商管理模块
- 修改翻译引擎模块，解耦模型依赖
- 可能需要重构现有的模型初始化代码

**配置影响**:
- 新增配置文件或环境变量支持多模型配置
- 需要存储多个 API Key 和配置信息

**依赖影响**:
- 可能需要添加新的 SDK 依赖（DeepSeek SDK 或通用的 OpenAI 兼容 SDK）
- 需要统一不同提供商的 API 调用接口

**用户影响**:
- 用户可以选择不同的模型进行翻译
- 降低使用成本（DeepSeek 性价比更高）
