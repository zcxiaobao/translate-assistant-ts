# translation-engine

## Purpose

翻译引擎负责在产品经理和开发工程师之间进行双向翻译，利用 AI 将产品语言转换为技术语言，或将技术语言转换为业务语言。

## Requirements

### Requirement: 支持产品到开发的翻译
系统必须能够将产品经理的需求描述翻译成开发工程师可理解的技术语言，包含技术实现方案、数据来源、性能要求和工作量评估。

#### Scenario: 产品经理输入需求描述
- **WHEN** 用户选择"产品→开发"翻译方向并输入产品需求描述
- **THEN** 系统生成包含以下内容的技术翻译：推荐的技术实现方案、数据来源和处理方式、性能和实时性要求、预估开发工作量

#### Scenario: 产品需求缺少关键信息
- **WHEN** 用户输入的产品需求描述较为模糊或缺少技术细节
- **THEN** 系统在翻译结果中主动补充技术关注点和需要明确的问题

### Requirement: 支持开发到产品的翻译
系统必须能够将开发工程师的技术方案翻译成产品经理可理解的业务语言，包含用户体验影响、业务增长空间和商业价值。

#### Scenario: 开发工程师输入技术方案
- **WHEN** 用户选择"开发→产品"翻译方向并输入技术实现描述
- **THEN** 系统生成包含以下内容的业务翻译：对用户体验的实际影响、支持的业务增长空间、成本降低的商业价值

#### Scenario: 技术方案缺少业务关联
- **WHEN** 用户输入的技术方案仅描述技术细节而缺少业务背景
- **THEN** 系统在翻译结果中主动补充业务价值和用户影响的分析

### Requirement: 集成 AI 服务
系统必须从 LLM 提供商管理器获取当前激活的 AI 提供商，用于执行翻译逻辑，支持动态切换提供商而无需重启服务。

#### Scenario: 成功调用 AI API
- **WHEN** 系统接收到翻译请求
- **THEN** 系统从 LLM 提供商管理器获取当前激活的 Provider，调用其 API 并返回翻译结果

#### Scenario: AI API 调用失败
- **WHEN** AI API 调用失败（网络错误、API 错误、超时等）
- **THEN** 系统返回友好的错误提示，告知用户调用失败原因

#### Scenario: 当前激活的提供商未配置有效的 API Key
- **WHEN** 当前激活的提供商未配置有效的 API Key
- **THEN** 系统提示用户配置该提供商的 API Key 并拒绝执行翻译

#### Scenario: 动态切换提供商
- **WHEN** 在运行时切换到另一个 LLM 提供商（如从 OpenAI 切换到 DeepSeek）
- **THEN** 后续的翻译请求使用新激活的提供商，无需重启服务

#### Scenario: 提供商管理器未初始化
- **WHEN** 系统尚未初始化任何 LLM 提供商
- **THEN** 系统抛出错误，提示需要先注册并激活至少一个提供商

### Requirement: 支持流式输出
系统必须支持流式输出，实时展示 AI 生成的翻译结果，而非等待全部内容生成后再返回。

#### Scenario: 流式返回翻译结果
- **WHEN** AI API 开始生成翻译内容
- **THEN** 系统实时向客户端推送每个生成的文本片段

#### Scenario: 流式输出中断
- **WHEN** 流式输出过程中发生错误（如网络中断）
- **THEN** 系统停止推送并返回已生成的部分内容和错误信息

### Requirement: 提供可插拔的 AI 提供商接口
系统必须设计可插拔的 AI 提供商接口，支持未来扩展到其他 AI 服务（如 Claude、国内大模型）。

#### Scenario: 切换 AI 提供商
- **WHEN** 管理员通过配置切换 AI 提供商（如从 OpenAI 切换到 Claude）
- **THEN** 系统使用新的 AI 提供商执行翻译，无需修改业务逻辑代码

#### Scenario: 添加新的 AI 提供商
- **WHEN** 开发者需要添加新的 AI 提供商支持
- **THEN** 开发者只需实现统一的接口，无需修改核心翻译逻辑
